# Руководство пользователя MKG

`http://localhost:8000` · после обновления: **Ctrl+F5**

## Навигация

| Вкладка | Назначение |
|---------|------------|
| **Главная** | Загрузка, строка этапа проекта |
| **Граф** | Компактная карта (сущности + межсл. связи) |
| **Полный граф** | Все узлы, включая TextParagraph |
| **Qdrant** | Индексация, список точек, лог (как Neo4j) |
| **Настройки** | Модели LLM, OCR, embedding doc/query |

Слева — **Документы** (на всех вкладках кроме Главной/Настроек).

## Превью документа

После выбора документа слева появляется **панель превью**:

- **▼** — развернуть: Markdown, поиск, логи пайплайна
- **Построить граф** — extraction → переход на «Граф»
- **↺ OCR** — переобработка ingestion
- **Компактный / Полный / Qdrant / Neo4j ↗**

## Граф

- Выбор документа в toolbar или слева
- **Перетаскивание узлов**, лёгкая физика
- **Изменение высоты**: потяните нижний край canvas или угол (CSS resize)
- **Список узлов** — кнопка в toolbar (скрыт по умолчанию)
- Фильтры L1–L6, «Межслойные», «Связи»

## Qdrant

1. Выберите документ
2. **Индексировать** — Yandex → `mkg_chunks` / `mkg_claims`
3. **Точки в Qdrant** — список как узлы Neo4j (node_id, layer, текст)
4. **Лог индексации** — под кнопками

API: `GET /api/v1/agents/documents/{id}/embeddings/points`

## Настройки

На вкладке **Настройки** задаётся модель для каждого сервиса:

| Сервис | Поле | Назначение |
|--------|------|------------|
| Извлечение графа | LLM | Extraction L1–L6 + фильтр чанков ingestion |
| OCR | OCR | PDF/изображения (`auto` = по типу файла) |
| Qdrant индексация | Embedding doc | Запись точек в коллекции |
| Qdrant поиск | Embedding query | Семантический search |

API: `GET/PUT /api/v1/config/models`

Сохранение в Postgres (`runtime_config`), применяется к **новым** задачам worker.
