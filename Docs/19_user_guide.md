# Руководство пользователя MKG

`http://localhost:8000` · UI cache `?v=137` · после обновления: **Ctrl+F5**

> **MVP без серверной аутентификации.** Роль выбирается в UI; API доступен на localhost без login. Не используйте публичный деплой без auth.

## Навигация

| Вкладка | Назначение |
|---------|------------|
| **Чат** | Роль + Быстрый/Подробный; история чатов; граф контекста; trace; источники; экспорт |
| **Документы** | Upload; **Пайплайн \| Markdown \| Граф**; «Все документы»; retry по этапам |
| **Qdrant** | Корпус L3+L4; N/M документов; поиск; карта кластеров; chip-фильтры |
| **Настройки** | LLM, OCR, embedding; обзор знаний; подписки; очистка базы |
| **Документация** | Встроенные md-разделы по архитектуре |

## Пайплайн документа (режим full)

```
Upload → OCR → Markdown (clean + marked) → Extraction L1–L6 → Neo4j → Qdrant → L4 HDBSCAN
```

| Этап | Что происходит |
|------|----------------|
| OCR | Yandex Vision / парсеры; fallback PyMuPDF |
| MD | **clean** — текст; **marked** — L3-маркеры + узлы графа |
| Graph | LLM извлекает L1–L6 в JSON |
| Neo4j | MERGE в граф знаний |
| Qdrant | `mkg_chunks` (L3) + `mkg_claims` (L4) |
| L4 | HDBSCAN: кластеры и аномалии на фактах |

На вкладке **Пайплайн** — статус каждого этапа и кнопки **↺** для перезапуска.

### Retry и reprocess-full

| Кнопка | Действие |
|--------|----------|
| ↺ OCR | `POST .../reprocess` — повтор ingestion |
| ↺ Извлечение | Повтор extraction |
| ↺ Neo4j | Повтор sync в Neo4j |
| ↺ Индекс | Индексация Qdrant |
| ↺ HDBSCAN L4 | Кластеризация L4 |
| **↺ Построить полный граф** | `POST .../reprocess-full` для документов `answers_only` → full pipeline |

## Вкладки документа

1. **Пайплайн** — цепочка этапов + слои L1–L6 (статус running/done).
2. **Markdown** — «Без разметки» / «С разметкой»; **Скачать .md**.
3. **Граф** — vis-network; фильтры L1–L6; **Расширенные фильтры**; компактный / полный вид.

**Все документы** — объединённый граф корпуса (кнопка в списке документов). Те же фильтры и панель **Сравнение** (2–3 Process/Material).

Шапка документа: **Построить граф**, **↺ OCR**, **Markdown**, **Логи**, **Neo4j ↗**.

## Чат

### Роль и скорость

- Выберите **карточку роли** (8 штук) — стиль ответа и права.
- Переключатель **Подробный · с рассуждениями** | **Быстрый** над полем ввода.
- **Нет pill-переключателя AI-режимов** (Оркестратор/Аудит/…) — оркестратор включается автоматически в «Подробном» для ролей с `can_run_agents`.

| Режим | Поведение |
|-------|-----------|
| **Быстрый** | `POST /chat/complete` с `speed_mode=fast`: unified search L1+L3+L4 → короткий LLM (~3–5 с) |
| **Подробный** | Async оркестратор L1–L6 (`/agents-service/run/async` + poll) или RAG-fallback |

> **Приветствия и мета-вопросы** («привет», «кто ты?») распознаются `query_classify` и
> отвечаются напрямую (пайплайн `direct`), без запуска поиска по корпусу.

### История чатов

- **+ Новый чат** — thread в Postgres.
- **Заголовок** — из первого сообщения пользователя (до ~55 символов); placeholder «Новый чат» заменяется автоматически.

### Загрузка файла (кнопка прикрепления)

| Вариант | Пайплайн |
|---------|----------|
| **Загрузить и обработать** | Полный: OCR → MD → граф → Neo4j → Qdrant → L4 |
| **Только для ответов** | OCR → MD → Qdrant (без графа и L4); позже — **reprocess-full** |

### Ответы

- **Структура** — разделы `## Сводка`, `## Источники по группам`, … (см. [`25_analytics_synthesis.md`](25_analytics_synthesis.md)).
- **Секции** — collapsible `<details>`; «Сводка» открыта, остальные свёрнуты.
- **Пояснить** — у каждого раздела и под всем ответом; отправляет уточняющий запрос.
- **Обновить** — повтор генерации на тот же вопрос пользователя.
- **Источники** — блок `<details>` **свёрнут по умолчанию**; chip → Markdown документа.
- **Экспорт** — MD, JSON-LD, Печать/PDF.
- **Граф контекста** — справа; в «Подробном» обновляется **live** при poll оркестратора.

### Trace

- **Подробный + оркестратор** — `orchestrator_init` → router ↔ `l*_agent` (раунды, шина) → discover → gap → synthesize.
- **Fallback / engineer** — `qdrant_search` → `graph_traversal` → layer trace → `llm_compose`.
- **Быстрый** — минимальный свёрнутый trace.

## Qdrant

Работа **по всему корпусу** — семантический поиск без обязательного фильтра по одному документу.

1. Статистика **N / M документов** в Qdrant + точки L3/L4.
2. **Индексировать** — per-doc или **Все документы**.
3. **Семантический поиск** — `POST /api/v1/search` (unified L1+L3+L4) по корпусу; chip-фильтры (тип док., L1/L3/L4, min score).
4. **Карта точек** — L4-кластеры; **клик по кластеру** → панель с составом и кнопками «Открыть кластер» / «Ближайший кластер».
5. **L4 cluster** — глобальный HDBSCAN.

## Настройки

| Секция | Назначение |
|--------|------------|
| Модели | LLM, OCR, embedding doc/query (`GET/PUT /config/models`) |
| **Доступ к данным** | Матрица role × гриф (`GET/PUT /settings/data-access`, PUT — только admin) |
| Обзор знаний | Dashboard: документы, аномалии, противоречия, домены, зоны риска |
| Подписки | Watchlist ключевых слов → toast при новом документе |
| Диагностика | Очистка базы (dev only) |

> **Доступ к данным.** Роль пользователя ограничивает видимые документы по грифу
> (`открытый` / `внутренний` / `конфиденциальный` / `строго`). Полный доступ — только у `admin`.
> `viewer` видит лишь `открытый`. Скрытые документы отображаются счётчиком «🔒 N скрыто».

## Очистка базы

**Настройки** → **Диагностика и очистка** → **Очистить базу**  
или `POST /api/v1/admin/clear?confirm=true` (только dev).
