# Руководство пользователя MKG

`http://localhost:8000` · UI cache `?v=43` · после обновления: **Ctrl+F5**

## Навигация

| Вкладка | Назначение |
|---------|------------|
| **Чат** | Роль + AI-режим; 📎 upload; trace; источники; «Сохранить как MD» |
| **Документы** | Upload; **Пайплайн \| Markdown \| Граф**; retry по этапам |
| **Qdrant** | L3/L4 индексация, поиск, карта точек, L4-кластеры |
| **Настройки** | LLM, OCR, embedding doc/query; очистка базы |

## Пайплайн документа (режим full)

```
Upload → OCR → Markdown (clean + marked) → Extraction L1–L6 → Neo4j → Qdrant → L4 HDBSCAN
```

| Этап | Что происходит |
|------|----------------|
| OCR | Yandex Vision / парсеры; fallback PyMuPDF |
| MD | **clean** — текст; **marked** — L3-маркеры + узлы графа |
| Graph | LLM извлекает L1–L6 в JSON |
| Neo4j | MERGE в граф знаний |
| Qdrant | `mkg_chunks` (L3) + `mkg_claims` (L4) |
| L4 | HDBSCAN: кластеры и аномалии на фактах |

На вкладке **Пайплайн** — статус каждого этапа и кнопки **↺** для перезапуска.

### Retry по этапам

| Кнопка | Действие |
|--------|----------|
| ↺ OCR | Повтор ingestion |
| ↺ Извлечение | Повтор extraction |
| ↺ Neo4j | Повтор sync в Neo4j |
| ↺ Индекс | Индексация Qdrant |
| ↺ HDBSCAN L4 | Кластеризация L4 |

## Вкладки документа

1. **Пайплайн** — цепочка этапов + слои L1–L6 (статус running/done).
2. **Markdown** — «Без разметки» / «С разметкой»; **Скачать .md**.
3. **Граф** — vis-network; фильтры L1–L6; компактный / полный вид.

Шапка документа: **Построить граф**, **↺ OCR**, **Markdown**, **Логи**, **Neo4j ↗**.

## Чат

### Роли

Выберите карточку роли (8 штук). Примеры:
- **Исследователь** — гипотезы, обзоры.
- **Аналитик** — поиск и графики.
- **Охотник за аномалиями** — L4 outliers.
- **Инженер** — пайплайн данных (без LangGraph).
- **Администратор** — настройки и очистка.

Промпт роли можно редактировать в панели «Промпт роли».

### AI-режимы

| Режим | Когда использовать |
|-------|-------------------|
| **Диалог** | Быстрый ответ: Qdrant L3+L4 + обход графа + LLM |
| **Гипотезы** | Новые идеи из фактов |
| **Аудит** | Проверка противоречий |
| **Аномалии** | L4 HDBSCAN + обход графа |
| **Обзор** | Литературный обзор |
| **Советы** | Рекомендации |

Режимы LangGraph доступны ролям с `can_run_agents=true`.

### Загрузка файла (📎)

Модальное окно:

| Вариант | Пайплайн |
|---------|----------|
| **Загрузить и обработать** | Полный: OCR → MD → граф → Neo4j → Qdrant → L4 |
| **Только для ответов** | OCR → MD → Qdrant (без графа и L4) |

Карточка загрузки показывает прогресс; **Открыть MD** — после готовности.

### Ответы

- **Trace** — шаги reasoning (поиск, граф, LLM или agent nodes).
- **Источники** — клик → Markdown документа.
- **Сохранить как MD** — экспорт ответа + источников.

## Qdrant

1. Фильтр по документу.
2. **Индексировать** — embeddings в `mkg_chunks` / `mkg_claims`.
3. **Семантический поиск** — запрос; hits с layer L3 или L4.
4. **Точки** — список / карта; `cluster_id`, `is_anomaly` для L4.
5. **L4 cluster** — запуск HDBSCAN.

## Настройки

| Сервис | Поле | Назначение |
|--------|------|------------|
| Извлечение | LLM | Extraction L1–L6 |
| OCR | OCR | PDF/изображения |
| Qdrant index | Embedding doc | Запись точек |
| Qdrant search | Embedding query | Семантический search |

`GET/PUT /api/v1/config/models` — сохранение в Postgres, применяется к новым задачам.

## Очистка базы

**Настройки** → **Диагностика и очистка** → **Очистить базу**  
или `POST /api/v1/admin/clear?confirm=true` (только dev).
