## 1. Цели системы (ТЗ §2)

| Цель ТЗ | Как решено в MKG | Статус |
|---------|------------------|--------|
| Агрегировать разнородные источники (отчёты, статьи, патенты, протоколы) | Upload + OCR/парсеры 11 форматов → единый Markdown | ✅ |
| Извлекать и связывать сущности, параметры, решения | LLM-extraction L1–L6 + `_bridge_text_to_layers` → Neo4j | ✅ |
| Семантический поиск по многомерным критериям | Qdrant (L1/L3/L4) unified search + Neo4j traversal + фильтры (материал/процесс/гео/время) | ✅ |
| Автоматически выявлять противоречия и пробелы | L4 HDBSCAN аномалии ✅ · движок Contradiction/KnowledgeGap 🟡 | 🟡 |
| Поддержка инженерных решений с технико-экономическим и эко-обоснованием | Слой L6 (TechnologySolution, Economic/Environmental Indicator) + панель «Сравнение» | ✅ |

## 2. Функциональные требования

### 2.1 Импорт и нормализация (ТЗ §3.1)

| Требование | Реализация | Статус |
|------------|------------|--------|
| Загрузка MD/PDF/DOCX (+ таблицы, формулы) | `POST /api/v1/documents`; форматы: PDF, PNG/JPG/WEBP/BMP/TIFF, MD, TXT, CSV, JSON, YAML, XML, DOCX, XLSX | ✅ |
| Определение типа документа (статья/патент/отчёт/диссертация) | LLM-классификация в extraction (`doc_type`) | ✅ |
| Разбивка на абзацы/таблицы/заголовки + эмбеддинги на чанк | L3: TextParagraph, HeadingContext; эмбеддинги в Qdrant `mkg_chunks` | ✅ (TableMatrix 🟡) |
| Метаданные: автор, дата, гриф, организация, гео | `classification`, `geography`, `source_location`, `material_date`; L2 Expert/Organization/Location | ✅ |
| Ru/En + автоопределение языка | `langdetect`, L3 LangContext, двуязычные поля `*_ru/*_en` | ✅ |
| Ремонт кодировки / «мусорных» чанков | `ftfy`, эвристики + LLM chunk-filter | ✅ |
| Фото → диаграмма (vision-агент) | OCR-путь есть; отдельный vision→Mermaid агент — не доведён | ⬜ |

### 2.2 NLP-извлечение сущностей и связей (ТЗ §3.2)

| Требование | Реализация | Статус |
|------------|------------|--------|
| Распознавание сущностей (материалы, реагенты, процессы, оборудование, параметры, эксперты, организации) | L1 (Material, Process, Equipment, ChemicalReagent, StandardMetric), L2 (Expert, Organization, Location, Timeline) | ✅ |
| Извлечение связей («процесс → использует материал» и др.) | Типизированные рёбра `:USES_MAT`, `:OPERATES_PROC`, `:SHOWED_EFFECT`, `:AUTHORED`, … | ✅ |
| Числовые условия и диапазоны (концентрации, T, скорости, экономика) | L4 Measurement (`numeric_value`, `unit`, `measurement_error`), StandardMetric диапазоны | ✅ |
| Синонимы/жаргон → онтология (SynonymMap) | Alias-поля в L1 + `search_query_variants`; узел SynonymMap как таковой | 🟡 |
| Привязка факта к тексту + оценка уверенности | `:CONTEXT_FOR`, `:DATA_SOURCE_FOR` с `confidence` | ✅ |

### 2.3 Граф знаний (ТЗ §3.3 / раздел «Построение графа»)

| Требование | Реализация | Статус |
|------------|------------|--------|
| Онтология домена (6 слоёв) | `packages/core/src/mkg_core/ontology.py`; схема `packages/graph/schema.cypher` | ✅ |
| Хранение графа + сложные обходы (Cypher) | Neo4j 5, `graph_traversal.py` (`GRAPH_TRAVERSAL_MAX_HOPS`) | ✅ |
| Версионирование фактов | Measurement `version`/`valid_from`/`valid_to`, `:PREVIOUS_VERSION` (в модели) | 🟡 |
| Идемпотентная загрузка | `loader.py` — `MERGE` по `id` | ✅ |

### 2.4 Поиск и навигация (ТЗ §3.4)

| Требование | Реализация | Статус |
|------------|------------|--------|
| Семантический поиск на естественном языке | `POST /api/v1/search` (unified L1+L3+L4) + чат `POST /chat/complete` | ✅ |
| Многоуровневая фильтрация (тип/материал/гео/год/достоверность) | query facets, chip-фильтры UI, фильтр по `document_ids`, гео/год | ✅ |
| Числовые диапазоны | L4 Measurement + StandardMetric `:EVALUATED_AGAINST` | 🟡 |
| Сравнительные запросы (RU vs зарубеж, вариант А vs Б) | `geography` фильтр + панель «Сравнение» (2–3 Process/Material), `compare_api` | ✅ |
| Визуализация графа (цепочки материал→процесс→оборуд→результат) | vis-network, фильтры L1–L6, «Все документы» | ✅ |
| Подсветка противоречий/пробелов | L4-аномалии на карте кластеров; полный Contradiction UI | 🟡 |
| Связанные эксперты/лаборатории | L2 Expert/Organization в графе и ответах | ✅ |

### 2.5 Аналитика и синтез (ТЗ §3.5)

| Требование | Реализация | Статус |
|------------|------------|--------|
| Структурированные ответы (лит-обзор, группировка) | Секции `## Сводка / ## Источники по группам / ## Пробелы`; `25_analytics_synthesis.md` | ✅ |
| Консенсус vs разногласия, число подтверждающих источников | Группировка источников + trace; полный consensus-scoring | 🟡 |
| Степень уверенности в ответе | confidence в extraction + retrieval factors; единый composite score | 🟡 |
| Выявление пробелов (неизученные комбинации) | KnowledgeGap (модель) + подсказки в синтезе; авто-детектор | 🟡 |
| Рекомендации (похожие кейсы, эксперты, смежные темы) | LangGraph mode `recommendation` / `hypothesis` | ✅ |

## 5. Эталонные запросы ТЗ (проверка «на живых вопросах»)

| Запрос из ТЗ | Что задействуется | Ожидаемый результат |
|--------------|-------------------|---------------------|
| Методы обессоливания воды по концентрациям сульфатов/хлоридов и сухому остатку ≤1000 мг/дм³ | L1 Process/Material + L4 Measurement + StandardMetric + числовые фильтры | Список решений с параметрами и источниками |
| Циркуляция католита при электроэкстракции никеля, оптимальная скорость потока | L1 Process/Equipment + L4 Measurement + L3 цитаты | Технические решения + скорость + источник |
| Эксперименты/публикации по Au, Ag, МПГ (штейн/шлак) за 5 лет | L2 Document/Timeline + гео/год фильтр + семантический поиск | Список работ, сгруппированный по годам/гео |
| Закачка шахтных вод в глубокие горизонты (RU vs зарубеж) + ТЭП | `geography` фильтр + L6 EconomicIndicator + «Сравнение» | Сравнительная таблица RU/зарубеж |

> Практическая заметка для демо: наполнение корпуса влияет на полноту ответов. Проверяйте на
> сущностях, реально присутствующих в загруженных документах (например, «никель», «Outokumpu»).

## 6. Технологические рекомендации ТЗ vs выбор MKG

| Рекомендация ТЗ | Выбор MKG | Обоснование |
|-----------------|-----------|-------------|
| Графовая БД: Neo4j / Neptune / JanusGraph | **Neo4j 5** | Cypher, зрелость, локальный контейнер |
| Векторный поиск / движок | **Qdrant** (`query_points`) | Быстрый ANN, payload-фильтры, 3 коллекции L1/L3/L4 |
| NLP: DeepPavlov / spaCy / ruBERT | **LLM-first (YandexGPT 5.x)** + rule-based fallback | Гибкость по доменам, ru/en, JSON-разметка слоёв |
| Онтологии: OWL/RDF/SHACL | Собственная типизированная онтология (`ontology.py`) | Прагматичный MVP под Neo4j |
| FAIR-принципы | Источник на каждый факт, версионирование, метаданные, экспорт | Findable/Accessible/Interoperable/Reusable (частично) |
